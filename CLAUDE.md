## Project Overview

A collection of hands-on demonstrations showing how to use [CUDA Python](https://github.com/nvidia/cuda-python) to accelerate common machine learning algorithms directly on NVIDIA GPUs. Targeted at ML practitioners and researchers who want low-level GPU control without leaving Python, bridging the gap between high-level frameworks and raw CUDA programming.

## Architecture

- **Backend**: Python 3.11+
- **Frontend**: CLI / Jupyter notebooks
- **Database**: None
- **Background Jobs**: None
- **Deployment**: Local only (requires NVIDIA GPU + CUDA 12.x)

## Key Technologies

- `cuda-python` — NVIDIA's official Python bindings for CUDA (cuda.core + cuda.bindings)
- `numpy` — CPU baseline reference implementations
- `cupy` — Optional GPU array interop
- `torch` — Optional PyTorch interop demonstrations
- `pytest` — Testing framework
- `ruff` — Linting and formatting
- `jupyter` — Interactive notebook demos

## API Structure

Each demo follows this interface contract:

```python
# Every demo module exposes:
def run_demo(n_samples: int = 10_000, seed: int = 42) -> BenchmarkResult: ...
def run_cpu_baseline(data: np.ndarray, **kwargs) -> tuple[Any, float]: ...
def run_gpu_version(data: np.ndarray, **kwargs) -> tuple[Any, float]: ...

# BenchmarkResult dataclass
@dataclass
class BenchmarkResult:
    cpu_time_s: float
    gpu_time_s: float
    speedup: float
    outputs_match: bool  # correctness check vs CPU
```

## Data Models

- `BenchmarkResult` — timing + correctness for one CPU vs GPU comparison
- `DeviceInfo` — CUDA device properties (from `cuda.core`)
- `KernelModule` — compiled CUDA kernel wrapper

## Key Workflows

1. **Demo execution**: load data → run CPU baseline → run GPU version → compare outputs → print speedup
2. **Kernel launch**: compile PTX via `cuda.core.Program` → allocate device memory → launch grid → copy back → free
3. **Interop**: allocate with CUDA Python → wrap as CuPy/PyTorch tensor → operate in-place → verify

## Security Considerations

- All demos run fully locally — no network access, no external APIs
- No user input is passed to shell commands or eval
- GPU memory must be explicitly freed to avoid leaks; use context managers where possible

## Important Notes

- **GPU required**: Core demos cannot be meaningfully tested without an NVIDIA GPU
- **CUDA 12.x**: `cuda-python` requires CUDA Toolkit 12.x installed system-wide
- **cuda.core vs cuda.bindings**: Prefer `cuda.core` (high-level) for new code; drop to `cuda.bindings` only when direct C API access is necessary
- **Memory discipline**: Always free device allocations; unfreed GPU memory persists across Python objects
- **Correctness first**: Every GPU implementation must be validated against its NumPy baseline before reporting speedups
