## Summary

Create a top-level `README.md` that serves as the user-facing entry point for the CUDA Python ML Demos project. It should enable a new user to understand the project's purpose, install dependencies, run their first demo, and navigate to deeper resources — all within 5 minutes of cloning the repo.

---

## Background

The project has grown to 8 demo directories, 12 backend variants, 37 slide documentation files, and (by Sprint 9) Jupyter notebooks. New visitors to the repository have no top-level README to orient them. The existing `CLAUDE.md` is written for AI agents and developers, not end users. A polished `README.md` is essential for discoverability, usability, and credibility.

## API / Interface Design

```markdown
# CUDA Python ML Demos

[![CI](https://github.com/owner/repo/actions/workflows/ci.yml/badge.svg)](...)

> Hands-on demonstrations of GPU-accelerated ML algorithms using NVIDIA's cuda-python.

## Prerequisites
- Python 3.11+
- NVIDIA GPU + CUDA Toolkit 12.x (for GPU demos)
- `pip install cuda-python numpy`

## Quickstart (CPU-only — no GPU required)
\`\`\`bash
git clone https://github.com/owner/repo
cd cuda-python-ml-demos
pip install numpy
python demos/09_kernel_model/main.py
\`\`\`

## Demo Overview
| Demo | Description | Needs GPU |
|------|-------------|-----------|
| 01_core_apis | CUDA Python API basics ... | Yes |
| 02_kmeans | GPU k-means clustering | Yes |
...

## Running Tests
\`\`\`bash
pytest tests/ -m "not gpu"   # 72 CPU tests, no GPU needed
pytest tests/ -v              # all tests (requires GPU)
\`\`\`
```
