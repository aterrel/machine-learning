## Summary

Demonstrate and document GPU memory management patterns that matter for ML: allocation strategies, memory pools, pinned host memory for fast transfers, stream-ordered allocation, and avoiding fragmentation — with concrete examples relevant to iterative training workloads.

---

## Background

Memory management is one of the most common sources of bugs and performance problems in GPU ML code. ML practitioners who use high-level frameworks rarely see these issues because the framework manages memory. CUDA Python exposes this layer directly, which is powerful but requires explicit management. This demo covers the patterns users need to manage memory correctly and efficiently.

## Notes

- Pinned memory is page-locked — OS cannot swap it out — enables DMA transfers that bypass the CPU
- Stream-ordered allocation (`cudaMallocAsync`) avoids global allocator lock — important for concurrent ML workloads
- Memory fragmentation is a real concern in iterative training: pre-allocation avoids it
