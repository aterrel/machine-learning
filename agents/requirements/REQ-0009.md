# REQ-0009: Slide-Based Demo Documentation

**Status**: Active
**Priority**: P1
**Objective Reference**: Objective 6 — Provide clear documentation and educational materials for ML practitioners learning GPU programming with CUDA Python
**Created**: 2026-05-01
**Author**: Product Manager
**Assigned To**: Programmer

---

## Summary

Create `docs/slides/` — a directory of Markdown slide files, one per concept, that walk through each demo in the repository. Each slide is self-contained: it names the concept, explains it in plain language, shows a representative code snippet with a file:line reference, and fits on a single screen (≤60 lines). The slide set covers all 8 demo directories plus an introduction section. No presentation software is required — slides are plain Markdown files usable in any editor, Jupyter, or static site generator.

---

## Background

The demos directory contains 8 algorithm directories with 3–5 implementation files each. A new user exploring the repo faces a wall of Python files with no narrative thread. The slide files provide a guided tour: they introduce each concept at the right level of abstraction, highlight the single most important pattern in each file, and link back to the source so readers can follow up. The slide-per-concept granularity (not file-per-demo) matches how practitioners actually learn — one idea at a time.

## Notes

- Slides are documentation, not tests. No test coverage is required for this REQ.
- The `**Source:**` line format: `**Source:** \`demos/02_kmeans/gpu_kmeans.py:45–62\`` — use an em-dash range when the snippet spans multiple lines.
- Backend comparison slides should be the last slide in each section so readers see individual implementations before the side-by-side view.
- Line numbers in `**Source:**` references should be verified against the actual files at time of authoring.
