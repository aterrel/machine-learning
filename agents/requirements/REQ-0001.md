## Summary

Demonstrate `cuda.core` and `cuda.bindings` APIs through hands-on, runnable Python examples that cover the essential patterns needed for ML workloads: device initialization, memory allocation, kernel compilation, kernel launch, and result retrieval.

---

## Background

CUDA Python exposes two layers: `cuda.bindings` (raw C API bindings) and `cuda.core` (Pythonic high-level wrappers). Developers coming from NumPy or PyTorch may not know which API to use when, or how to set up a minimal CUDA Python program. This demo establishes the foundation all other demos build on.

## Acceptance Criteria

- [ ] AC-1: User can run `python demos/01_core_apis/main.py` and sees device name, compute capability, and total VRAM printed
- [ ] AC-2: User can inspect the demo source and identify the cuda.core equivalent for each raw bindings call shown
- [ ] AC-3: A vector-add kernel launches successfully and produces correct output (verified against NumPy)
- [ ] AC-4: Running the demo on a machine without a CUDA GPU prints a clear error and exits with code 1
- [ ] AC-5: After demo completes, no CUDA memory is leaked (run with `compute-sanitizer` or equivalent)
- [ ] AC-6: Demo completes in under 30 seconds on RTX 3080 or equivalent hardware
