# REQ-0008: Unified Multi-Backend Comparison Demo (cuML + All Backends)

**Status**: Active
**Priority**: P1
**Objective Reference**: Objective 4 — Side-by-side comparisons of CPU vs GPU implementations with timing and speedup metrics; Objective 5 — Integrate CUDA Python with popular ML libraries
**Created**: 2026-05-01
**Author**: Product Manager
**Assigned To**: Programmer

---

## Summary

Create `demos/08_comparison/` — a single demo that runs the same ML algorithm (k-means, PCA, linear regression, and naive Bayes) across all available backends (NumPy, cuda-python, CuPy, Numba, cuML) and prints a unified comparison table. This demo surfaces the entire GPU Python abstraction stack in one place and introduces cuML as the highest-level option. cuML code is always written but silently skipped if RAPIDS is not installed.

---

## Background

After REQ-0007 adds Numba and CuPy variants to each individual demo, users can compare backends per-demo — but they have to run multiple scripts to see the full picture. `demos/08_comparison/` answers the question *"which GPU Python tool should I use for this algorithm?"* in a single run. It also introduces cuML (RAPIDS Machine Learning), which is the scikit-learn-compatible GPU ML library: calling `cuml.cluster.KMeans` is one line of code and zero CUDA programming. Showing cuML alongside cuda-python makes it clear that cuML is the right choice when you don't need custom kernels.

## Notes

- cuML installation: RAPIDS recommends conda (`conda install -c rapidsai cuml=24.XX cuda-version=12.X`) or the RAPIDS Docker image. A pip path (`pip install cuml-cuXX`) exists for some versions. See https://rapids.ai/install for the version matrix.
- cuML's API mirrors scikit-learn: `model.fit(X)`, `model.predict(X)`, `model.transform(X)`. Inputs can be numpy arrays; cuML converts internally.
- The comparison demo intentionally does NOT re-implement algorithms — it calls existing demo modules. This ensures the comparison table reflects exactly the same implementations that users can inspect in the individual demo directories.
- Speedup column: `numpy_time / backend_time`. A "Correct" value of `--` for numpy (it is the reference) and `True/False` for GPU backends.
