## Summary

Demonstrate how CUDA Python memory can be shared zero-copy with NumPy (via `__cuda_array_interface__`), CuPy arrays, and PyTorch tensors, enabling users to drop CUDA Python into an existing ML pipeline without forced data copies.

---

## Background

Real ML workflows rarely use a single library in isolation. Users who adopt CUDA Python for custom kernels still want to pass results back to PyTorch for gradient computation or to CuPy for further GPU operations. Interoperability is what makes CUDA Python practical rather than a toy. This demo shows the zero-copy bridges that CUDA Python's memory objects expose.

## Notes

- `cuda_array_interface` spec: https://numba.readthedocs.io/en/stable/cuda/cuda_array_interface.html
- DLPack is the cross-framework standard; prefer it over library-specific mechanisms where available
- PyTorch's `torch.utils.dlpack.from_dlpack()` and CuPy's `cupy.from_dlpack()` are the recommended entry points
