## Summary

Implement GPU-accelerated versions of four foundational ML algorithms — k-means clustering, PCA, linear regression, and naive Bayes — using CUDA Python, each paired with a NumPy CPU baseline for correctness validation.

---

## Background

These algorithms are the workhorses of classical ML. They all involve large matrix operations or distance computations that are highly parallelizable on GPU. Implementing them with CUDA Python (rather than CuPy or cuML) shows users exactly how the GPU acceleration works at the kernel level, demystifying what higher-level libraries do under the hood.

## Acceptance Criteria

- [ ] AC-1: User can run `python demos/02_kmeans/main.py` and sees cluster assignments, inertia, and GPU vs CPU timing
- [ ] AC-2: K-means GPU output cluster centroids match CPU output within 1e-3 tolerance on the same seed
- [ ] AC-3: User can run `python demos/03_pca/main.py` and sees explained variance ratios and timing comparison
- [ ] AC-4: PCA GPU top-k eigenvectors match NumPy `eigh` output (up to sign flip) within 1e-4 tolerance
- [ ] AC-5: User can run `python demos/04_linear_model/main.py` and sees R² score and MSE for GPU vs CPU
- [ ] AC-6: Linear regression GPU coefficients match NumPy least-squares within 1e-5 tolerance
- [ ] AC-7: Each demo prints a summary line: `GPU: X.XXs | CPU: X.XXs | Speedup: X.Xx | Correct: True`
- [ ] AC-8: Running any demo with `--n-samples 100000` achieves ≥ 2x GPU speedup
