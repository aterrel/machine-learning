## Summary

Create a GitHub Actions CI/CD pipeline that automatically validates code quality (lint + format) and CPU-safe tests on every push and pull request, without requiring a GPU. A second workflow supports GPU test execution via manual trigger or self-hosted runner.

---

## Background

The project currently has `pyproject.toml` with ruff configuration and a full test suite (72 CPU-safe tests + 26 GPU tests), but no automated CI pipeline. The CI/Build health indicator has been Red since Sprint 1. ML practitioners who clone the repo expect a green CI badge and confidence that the code works. This requirement addresses the gap.

The core architectural challenge is that the project requires an NVIDIA GPU for its full test suite, but standard GitHub-hosted runners do not provide GPUs. The solution is a two-tier pipeline: CPU-only validation runs automatically; GPU tests are available on-demand.

## API / Interface Design

```yaml
# .github/workflows/ci.yml (abbreviated)
name: CI
on: [push, pull_request]
jobs:
  lint:
    runs-on: ubuntu-latest
    steps: [checkout, setup-python 3.11, pip install ruff, ruff check, ruff format --check]
  test-cpu:
    runs-on: ubuntu-latest
    steps: [checkout, setup-python 3.11, pip install numpy pytest, pytest -m "not gpu"]

# .github/workflows/gpu-ci.yml (abbreviated)
name: GPU CI
on:
  workflow_dispatch:
jobs:
  test-gpu:
    runs-on: [self-hosted, gpu]
    steps: [checkout, setup-python 3.11, pip install cuda-python numpy cupy torch pytest ruff, pytest -v]
```

## Notes

- Architecture design: ARCH-006
- `ruff` version should be pinned in `pyproject.toml` (or add `ruff>=0.4` to CI install) to avoid format drift
- The CPU CI job explicitly excludes `cuda-python` to ensure that any accidental top-level GPU import is caught as a `ModuleNotFoundError` at test collection time — this is a deliberate correctness gate
