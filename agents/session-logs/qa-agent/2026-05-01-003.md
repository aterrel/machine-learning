## Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Use `patch.dict("sys.modules", {"cupy": None})` to simulate missing CuPy/torch | Functions do `import cupy` inside function body; setting the key to `None` makes the import raise `ImportError` without reloading the module | Uninstalling the library (destructive); `importlib.reload()` (causes C-extension double-load issues on Python 3.14) |
| Import interop modules at test function call time via helper (`_load_cupy_interop()`) | Avoids importing cuda-python at pytest collection time; keeps CPU tests runnable without GPU | Module-level import with try/except (noisier, still requires GPU-aware env for collection) |
| 6 separate `__cuda_array_interface__` tests instead of 1 combined | Easier to diagnose which structural constraint failed; follows project test style from test_memory.py and test_device.py | One monolithic assertion test (harder to debug on failure) |
| GPU pipeline tests accept n=5_000 or n=10_000 rather than full n=100_000 default | Faster execution in CI; correctness is not sample-size-dependent | Using default n (unnecessary slowdown); using n=1 (less representative) |
