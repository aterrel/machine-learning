## Work Completed

### tests/test_memory.py (TL-004 fix from Sprint 1)

- 2 CPU-only tests: `test_alloc_pinned_import_error`, `test_free_pinned_import_error`
  - Each sets `sys.modules["cuda.bindings.cudart"] = None` directly to simulate absence
    of cuda-python, then verifies `RuntimeError` is raised from the local import inside
    `alloc_pinned` / `free_pinned`
  - The reload-based approach in the task brief was not viable on Python 3.14 because
    removing `src.utils.memory` from `sys.modules` caused numpy to attempt a second C extension
    load on re-import, which the interpreter rejects; direct `sys.modules` patching was used instead
- 2 GPU tests: `test_device_buffer_context_manager`, `test_query_device_memory_keys`
  - Both marked `@pytest.mark.gpu` and use the `gpu_device` fixture for auto-skip

### tests/test_pca.py

- 5 CPU-only tests covering `pca_cpu` from `demos.03_pca.cpu_pca`:
  - Output shapes (components, explained_variance, X_transformed)
  - Orthonormality of components (`components @ components.T ≈ I` within 1e-6)
  - Explained variance all strictly positive
  - Explained variance non-increasing order
  - Determinism (same input → identical components)
- 1 GPU test: `test_pca_gpu_matches_cpu` — sign-flip tolerant via `|diag(cpu @ gpu.T)| > 1 - 1e-3`
- Imports via `importlib.import_module("demos.03_pca.cpu_pca")` (numeric prefix workaround)

### tests/test_linear.py

- 4 CPU-only tests covering `linear_regression_cpu` from `demos.04_linear_model.cpu_linear`:

  - R² > 0.99 for low-noise data
  - MSE strictly positive
- 1 GPU test: `test_linear_gpu_matches_cpu` — weights within 1e-3
- Imports via `importlib.import_module("demos.04_linear_model.cpu_linear")`

## Decisions Made

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Direct `sys.modules` patching instead of importlib.reload for memory tests | Reloading on Python 3.14 caused numpy C extension double-load error | Using `importlib.reload` inside `patch.dict` (caused same error); monkeypatching the function directly (would not test the import path) |
| Sign-flip tolerant comparison for PCA GPU test | PCA eigenvectors are unique only up to sign; direct comparison would give false negatives | Requiring the GPU to match CPU exactly (incorrect); using absolute tolerance on components (would miss sign flips) |
| Import GPU demo modules inside GPU test functions | Avoids importing cuda-python at module collection time, keeps CPU-only tests runnable without GPU installed | Importing at module level with try/except (noisier; still requires cuda present for collection) |
