### demos/06_interop/ — NumPy/CuPy/PyTorch interop
Files created:
- `demos/06_interop/__init__.py` — empty
- `demos/06_interop/cupy_interop.py`
  - `demo_cuda_python_to_cupy`: allocates DeviceBuffer, H2D copies numpy array,
    builds `__cuda_array_interface__` dict, wraps pointer via `cupy.cuda.MemoryPointer`
    (zero-copy), runs `cp.sum` and verifies against numpy reference.
  - `demo_cupy_to_cuda_python`: creates CuPy array, extracts `arr.data.ptr`,
    passes integer pointer directly to a `scale_inplace` CUDA Python kernel.
  - Both functions guard on `import cupy` with a friendly skip message if absent.
- `demos/06_interop/torch_interop.py`
  - `demo_cuda_python_to_torch`: allocates DeviceBuffer, shows interface dict,
    bridges via CuPy → `torch.as_tensor()` (zero-copy) if CuPy available,
    otherwise falls back to D2H → `torch.from_numpy().cuda()` with a note.
  - `demo_torch_to_cuda_python`: creates PyTorch CUDA tensor, extracts
    `tensor.data_ptr()`, passes to `scale_inplace` kernel, verifies via `.cpu().numpy()`.
  - Both functions guard on `import torch` with a friendly skip message if absent.
- `demos/06_interop/pipeline.py`
  - `run_end_to_end_pipeline(n)`: five-step pipeline:
    1. H2D copy numpy → DeviceBuffer
    2. ReLU kernel in-place
    3. Optional CuPy: zero-copy wrap, compute L2 norm
    4. Optional PyTorch: wrap or D2H fallback, compute sum
    5. D2H and correctness check
  - Returns result dict with `cupy_available`, `torch_available`, `relu_correct`,
    `cupy_norm`, `torch_sum`.
- `demos/06_interop/main.py`
  - `main()`: checks GPU, reports library availability, runs all three sub-demos.
