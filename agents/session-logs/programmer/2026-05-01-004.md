### demos/05_naive_bayes/ — GPU Gaussian Naive Bayes

- `demos/05_naive_bayes/cpu_nb.py`
  - `gaussian_nb_cpu(X_train, y_train, X_test, n_classes)`:
    - Computes per-class log-priors, means, and variances (+ 1e-9 epsilon) on NumPy.
    - Log-likelihood computed in float64 for numerical stability; predictions via argmax.
    - Returns `(predictions, class_means, class_vars, log_priors)` so GPU path can
      reuse the fitted parameters directly.
  - Handles degenerate case where a class has zero training samples (uniform prior).

- `demos/05_naive_bayes/gpu_nb.py`
  - `gaussian_nb_gpu(X_train, y_train, X_test, n_classes)`:
    - **Fit on CPU**: same per-class statistics as `cpu_nb` (float32 for GPU transfer).
    - **Predict on GPU** via two custom CUDA kernels:
      1. `compute_log_likelihood`: one thread per (sample, class) pair; 2D grid
         (x-dim = sample blocks, y-dim = class index). Computes log N(x; mu, var)
         accumulated with log-prior into `log_probs[sample * n_classes + cls]`.
      2. `argmax_predictions`: one thread per test sample; scans `n_classes` log-probs
         and writes winning class index to `predictions[i]`.
    - Six `DeviceBuffer`s (X_test, means, vars, log_priors, log_probs, predictions)
      managed with a single `with` block for deterministic cleanup.
    - H2D / D2H via `cudart.cudaMemcpy`; stream synchronised before returning.
    - Returns `(predictions, class_means, class_vars, log_priors)`.

- `demos/05_naive_bayes/main.py`
  - CLI args: `--n-train` (5000), `--n-test` (1000), `--n-features` (16),
    `--n-classes` (4), `--seed` (42).
  - Data generation: `n_classes` cluster centers drawn from N(0, 3.0·I) via
    `np.random.default_rng(seed)`; points drawn from N(center, 0.5·I); both splits
    (train/test) shuffled.
  - Runs CPU baseline then GPU version; computes per-split accuracy against `y_test`.
  - Reports CPU vs GPU prediction agreement (should be 100% — same argmax on same
    fitted parameters with float32 inputs).
  - Prints `BenchmarkResult.summary_line()`.

## Design Notes

- **CPU fit / GPU predict split**: fitting touches the entire training set once with
  simple reductions — negligible compute, no benefit from GPU. Inference (evaluating
  the log-likelihood over all `(n_test × n_classes × n_features)` combinations) is
  the parallelisable bottleneck and maps cleanly to a 2D CUDA grid.
- **Exact prediction agreement**: CPU and GPU receive identical float32 means/vars/
  log_priors and identical float32 X_test, so the argmax should agree on every sample.
  Any divergence would indicate a kernel bug and is flagged with a WARNING.
- **Epsilon stability**: 1e-9 added to every class variance prevents log(0) in the
  log-likelihood kernel.
- All GPU paths guarded with `try/except ImportError` and `try/except RuntimeError`
  with `sys.exit(1)` and clear messages (consistent with other demos).
